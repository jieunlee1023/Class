
클래스 상속
-새로운 클래스를 정의 할 때
이미 구현된 클래스를 상속 받아서 
속성이나 기능을 확정하여 클래스를 구현

-이미 구현된 클래스보다
더 구체적인 기능을 가진 클래스를 구현해야 할 때 
기존 클래스를 상속함

상속 하는 클래스 : 상위 (부모parent, super) 클래스
상속 받는 클래스 : 하위 (자식child, sub) 클래스
A 클래스가 B클래스를 상속한다.
= B클래스가 A클래스를 상속 받는다.

(기능적으로 더 큰 클래스는 B(자식클래스),
B가 A의 모든 것을 그대로 물려받고,
자신의 기능을 추가하기 때문)

상위 클래스는 하위 클래스 보다 더 일반적인 개념과 기능을 가짐 : 기본김밥
하위 클래스는 상위 클래스 보다 더 구체적인 개념과 기능을 가짐 : 참치김밥, 치즈김밥
하위 클래스가 상위 클래스의 속성과 기능을 확장(extends)한다는 의미

차와 엔진은 상속과 어울리지 않는다.
차는 엔진을 포함하는 관계이다 (포함 관계)

오버라이드 <---> 오버로딩
상속은 bottom up 방식으로! ! ! 생각해야함

★ 중요 ★ 다형성 poltmorphism
(자바 프로그램의 꽃)
多(많은) 形(형태) , 다양한 형태로 바라 볼 수 있다.
ex) 배 (먹는 배, 사람 배, 타는 배)

-하나의 코드가 여러 자료형으로 구현되어 실행되는 것
-같은 코드에서 여러 다른 실행 결화가 나옴
- 정보 은닉, 상속과 더불어 객체 지향 프로그래밍의 가장 큰 특징 중 하나
- 다형성을 잘 활용하면 유연하고 확장성 있고,
유지보수가 편리한 프로그램을 만들수 있음


추상 클래스 (abstract class)

추상 클래스는 new 할 수 없음 (인스턴스화 할 수 없음)
